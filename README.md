# Курсы валют — десктоп-приложение с GUI и CLI

Приложение на Python, которое получает актуальные курсы валют с
открытого публичного API Центрального Банка РФ, позволяет
фильтровать и сортировать данные через графический интерфейс или
командную строку, а затем сохранять результат в JSON-файл.

## Возможности

- Получение актуальных курсов валют без регистрации и API-ключа.
- **Автоматическая загрузка курсов сразу при запуске** GUI — данные
  появляются в таблице, как только приложение откроется.
- Графический интерфейс на `tkinter` (входит в стандартную поставку
  Python), полностью на русском языке.
- **CLI-режим без GUI** (`--no-gui`) — получение, фильтрация,
  сортировка и сохранение курсов одной командой, удобно для
  скриптов, cron-задач и автоматизации.
- **Резервный кэш данных**: если сервер недоступен, приложение
  (и в GUI, и в CLI) показывает последние успешно полученные курсы
  из локального кэша с явной пометкой о том, что данные устарели.
- **Мгновенное применение фильтров** в GUI — любое изменение текста,
  диапазона курса, поля сортировки или направления обновляет таблицу
  без отдельной кнопки «Применить».
- Фильтрация по коду/названию валюты **регистронезависима**.
- Сортировка по коду, названию, курсу, изменению и изменению в
  процентах — по возрастанию или убыванию, в том числе кликом по
  заголовку колонки таблицы (▲/▼).
- Гибкая конфигурация через переменные окружения / файл `.env`
  (URL API, таймаут, путь к кэшу, путь сохранения по умолчанию).
- Логирование событий и понятные сообщения об ошибках сети.
- Источник данных абстрагирован через интерфейс `RateProvider` —
  замена API не требует изменений в GUI/CLI коде.

## Источник данных

Используется бесплатное зеркало официального API ЦБ РФ:
`https://www.cbr-xml-daily.ru/daily_json.js`. Регистрация, API-ключ
и авторизация не требуются — это полностью открытый источник.

## Установка

Требуется Python 3.9 или новее.

```bash
pip install -r requirements.txt
```

При желании скопируйте `.env.example` в `.env` и измените значения
под себя — это необязательно, без файла `.env` используются
значения по умолчанию.

## Запуск GUI

```bash
python main.py
```

После запуска:

1. Приложение **сразу же автоматически** загружает актуальные курсы
   с сервера ЦБ РФ. Если сервер недоступен — показываются последние
   сохранённые в кэше данные с предупреждением об их устаревании.
   Кнопка **«Обновить курсы»** позволяет повторить загрузку.
2. Любое изменение фильтров **применяется мгновенно**.
3. **«Сохранить в JSON»** сохраняет текущий отфильтрованный список.
4. Клик по заголовку колонки таблицы меняет сортировку (возрастание
   → убывание → отмена сортировки).

## Запуск в режиме командной строки (без GUI)

Полезно для автоматизации (cron, CI, серверные окружения без
графической подсистемы):

```bash
# Получить все курсы и сохранить в файл по умолчанию (currency_rates.json)
python main.py --no-gui

# Указать свой файл
python main.py --no-gui --output rates.json

# Фильтр по коду валюты (регистр не важен)
python main.py --no-gui --code usd --output usd.json

# Диапазон значений и сортировка по убыванию
python main.py --no-gui --min 10 --max 100 --sort value --desc
```

Параметры командной строки:

| Параметр    | Описание                                                       |
|-------------|-------------------------------------------------------------------|
| `--no-gui`  | запустить без графического интерфейса (обязателен для CLI-режима) |
| `--output`  | путь к итоговому JSON-файлу (по умолчанию `currency_rates.json`)  |
| `--code`    | подстрока для поиска по коду/названию валюты                      |
| `--min`     | минимальное значение курса                                         |
| `--max`     | максимальное значение курса                                         |
| `--sort`    | поле сортировки: `code`, `name`, `value`, `change`, `change_percent` |
| `--desc`    | сортировать по убыванию                                             |

Если сервер недоступен, CLI-режим, как и GUI, пытается использовать
локальный кэш и явно сообщает об этом в выводе.

### Поля фильтров GUI

| Поле                          | Описание                                                                 |
|--------------------------------|---------------------------------------------------------------------------|
| «Код или название валюты»     | Поиск подстроки по буквенному коду (`USD`) или названию («Доллар США»). Регистр не важен. |
| «Курс от» / «Курс до»          | Диапазон значения курса к рублю (включительно). Поддерживает точку и запятую как разделитель. |
| «Сортировать по»               | Поле сортировки: «Код валюты», «Название», «Курс», «Изменение», «Изменение, %». |
| «По убыванию»                  | Переключает направление сортировки между возрастанием и убыванием.        |

## Конфигурация

Все настройки можно переопределить через переменные окружения или
файл `.env` (см. `.env.example`):

| Переменная                   | Назначение                                          | Значение по умолчанию                          |
|-------------------------------|------------------------------------------------------|--------------------------------------------------|
| `CURRENCY_API_URL`            | URL источника данных о курсах                        | `https://www.cbr-xml-daily.ru/daily_json.js`      |
| `CURRENCY_REQUEST_TIMEOUT`    | Таймаут сетевого запроса (секунды)                    | `10`                                              |
| `CURRENCY_CACHE_FILE`         | Путь к файлу резервного кэша                          | `~/.currency_tracker_cache.json`                  |
| `CURRENCY_DEFAULT_OUTPUT`     | Путь сохранения по умолчанию для CLI                  | `currency_rates.json`                             |

## Формат сохраняемого JSON

```json
{
  "source_date": "2026-06-30T11:30:00+03:00",
  "saved_at": "2026-06-30T14:22:10",
  "count": 2,
  "rates": [
    {
      "char_code": "USD",
      "name": "Доллар США",
      "nominal": 1,
      "value": 90.1234,
      "previous": 89.5,
      "change": 0.6234,
      "change_percent": 0.6964
    },
    {
      "char_code": "EUR",
      "name": "Евро",
      "nominal": 1,
      "value": 98.0,
      "previous": 99.0,
      "change": -1.0,
      "change_percent": -1.0101
    }
  ]
}
```

| Поле             | Описание                                          |
|------------------|----------------------------------------------------|
| `source_date`    | дата актуальности курсов по данным ЦБ РФ           |
| `saved_at`       | время сохранения файла на локальной машине          |
| `count`          | количество записей в файле                          |
| `char_code`      | буквенный код валюты (ISO 4217)                     |
| `name`           | название валюты на русском языке                    |
| `nominal`        | номинал (за сколько единиц валюты указан курс)       |
| `value`          | текущий курс к рублю                                 |
| `previous`       | предыдущее значение курса                            |
| `change`         | абсолютное изменение курса                           |
| `change_percent` | изменение курса в процентах                          |

## Расширяемость источника данных

Получение данных абстрагировано через класс `RateProvider`
(`api_client.py`). Чтобы подключить другой источник курсов (другой
API, локальный файл, биржевые данные и т.д.), достаточно реализовать
новый класс с методом `fetch()`, возвращающим словарь
`{"date": str, "rates": List[CurrencyRate]}`, и передать его экземпляр
в `fetch_rates(provider=...)`. Остальной код (GUI, CLI, фильтры,
сохранение) менять не требуется.

## Тестирование

Проект покрыт unit-тестами всех ключевых модулей, включая получение
данных с замоканными сетевыми запросами (без обращения к реальному
серверу):

```bash
python -m unittest discover -s tests -v
```

Тесты охватывают: фильтрацию и сортировку, сохранение в JSON,
расчёт изменения курса, успешные и ошибочные сценарии получения
данных (таймаут, ошибка соединения, HTTP-ошибка, некорректный JSON,
отсутствующее поле), работу кэша и CLI-режим.

При каждом push и pull request в GitHub Actions автоматически


```bash

```

## Известные ограничения

- Для получения свежих данных требуется доступ в интернет; при его
  отсутствии используется локальный кэш последнего успешного ответа
  (если он есть).
- Используется один источник данных по умолчанию (ЦБ РФ); для
  использования альтернативного API нужно реализовать новый класс,
  унаследованный от `RateProvider`.
- Графический интерфейс рассчитан на десктоп (Windows/macOS/Linux)
  с установленным `tkinter`. На некоторых Linux-дистрибутивах его
  нужно установить отдельно: `sudo apt install python3-tk`. В
  окружениях без графической подсистемы используйте CLI-режим
  (`--no-gui`).
- Мгновенное применение текстового фильтра в GUI означает, что
  фильтрация запускается при каждом нажатии клавиши; так как список
  валют (несколько десятков записей) обрабатывается локально без
  сетевых запросов, заметных задержек это не вызывает.

## Чего в проекте намеренно нет

В проекте отсутствуют функции автоматической регистрации аккаунтов,
ротации/смены IP-адреса и сохранения cookie-файлов для обхода защит
сторонних сайтов. Используемый источник данных открытый и не требует
авторизации, поэтому такие механизмы не нужны и не реализовывались.
